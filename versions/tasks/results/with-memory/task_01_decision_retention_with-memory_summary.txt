We have decided to preserve the following compatibility constraints:

1. **Public CLI Unchanged**: The existing command-line interface (CLI) for adding, listing, and filtering todos must remain exactly as it is. No new CLI commands or argument structures will be added.
2. **Existing Exported Functions Unchanged**: Any functions that are currently exported (e.g., `add_todo`, `list_tasks`, `filter_tasks`) must not be renamed. Their signatures and behavior must remain identical.
3. **No New Imports Required**: The new filtering logic must not introduce any new external dependencies or imports that are not already present in the codebase.
