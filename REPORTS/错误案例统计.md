## 🐛 根本原因分析

### 1. 模型陷入工具调用死循环

**现象**:
- 模型不断重复调用相同的工具：`read_file`，参数为 `{"path":"todo"}`
- 工具返回错误：`error: invalid arguments for read_file: path is not a file`
- 模型看到错误后，仍然重复调用相同的工具，没有调整策略

**调试输出示例**:
```
[DEBUG] Model output: <tool>{"name":"read_file","args":{"path":"todo"}}</tool>
[DEBUG] Parsed: kind=tool, payload={'name': 'read_file', 'args': {'path': 'todo'}}
[DEBUG] Tool result: error: invalid arguments for read_file: path is not a file 
example: <tool>{"name":"read_file","args":{"path":"README.md","start":1,"end":80}}</tool>

[DEBUG] Model output: <tool>{"name":"read_file","args":{"path":"todo"}}</tool>
[DEBUG] Parsed: kind=tool, payload={'name': 'read_file', 'args': {'path': 'todo'}}
[DEBUG] Tool result: error: invalid arguments for read_file: path is not a file 
example: <tool>{"name":"read_file","args":{"path":"README.md","start":1,"end":80}}</tool>

... (重复 20 次)
```

### 2. 问题分类

**问题类型**: 模型能力限制，而非代码实现错误

**证据**:
1. ✅ **工具执行正确**: `run_tool()` 方法正确执行并返回错误信息
2. ✅ **错误信息清晰**: 工具返回了明确的错误信息和示例
3. ✅ **Memory Manager 正常工作**: 第 6 轮成功证明了约束保留功能正常
4. ❌ **模型无法从错误中学习**: 模型没有根据错误信息调整策略

### 3. 为什么第 6 轮成功了？

第 6 轮的任务是："Before finishing, remind me which compatibility constraints we decided to preserve."

**成功原因**:
- 这是一个**纯文本生成任务**，不需要工具调用
- 模型直接回答了问题，没有陷入工具调用循环
- **证明了 Memory Manager 的约束保留功能正常工作**：
  - 模型正确回忆了之前提到的约束：
    - Public CLI Unchanged
    - Existing Exported Functions Unchanged
    - No New Imports Required

## 💡 改进建议

### 1. 短期改进（代码层面）

#### 1.1 增强错误处理和重试机制

**当前问题**: 模型重复调用相同的错误工具

**改进方案**:
```python
def repeated_tool_call(self, name, args):
    """检测重复的工具调用"""
    tool_events = [item for item in self.session["history"] if item["role"] == "tool"]
    if len(tool_events) < 2:
        return False
    
    # 检查最近 3 次调用是否相同
    recent = tool_events[-3:]
    if len(recent) < 3:
        return all(item["name"] == name and item["args"] == args for item in recent)
    
    # 检查是否连续 3 次调用相同的错误工具
    if all(item["name"] == name and item["args"] == args for item in recent):
        # 检查是否都是错误
        if all("error:" in item.get("content", "") for item in recent):
            return True
    
    return False
```

#### 1.2 添加智能错误提示

**改进方案**: 在模型陷入循环时，提供更明确的指导


```python
class ToolCallAnalyzer:
    def analyze_pattern(self, history):
        """分析工具调用模式，检测循环"""
        recent_calls = [h for h in history if h["role"] == "tool"][-10:]
        
        # 检测重复错误
        error_pattern = self.detect_repeated_errors(recent_calls)
        if error_pattern:
            return {
                "type": "repeated_error",
                "tool": error_pattern["tool"],
                "count": error_pattern["count"],
                "suggestion": self.generate_suggestion(error_pattern)
            }
        
        return None
    
    def generate_suggestion(self, error_pattern):
        """生成改进建议"""
        tool = error_pattern["tool"]
        if tool == "read_file":
            return "Use list_files to see the directory structure first"
        # ... 其他工具的建议
```

## 🚀 解决方案 - 从 Ollama 切换到 DeepSeek API

### 问题回顾

使用 qwen3.5:2b (Ollama) 模型时遇到的问题：
- ❌ 模型在工具调用出错时陷入死循环
- ❌ 无法从错误信息中学习并调整策略
- ❌ 任务完成率低（仅 16.67%）

### 实施的解决方案

#### 1. 模型后端升级

**从 Ollama 本地模型 → DeepSeek API 云服务**

| 特性 | Ollama (qwen3.5:2b) | DeepSeek API |
|------|---------------------|--------------|
| 模型能力 | 2B 参数 | 更强的推理能力 |
| 工具调用处理 | 易陷入循环 | 更好的错误处理 |
| 部署方式 | 本地部署 | 云 API |
| 维护成本 | 需要管理本地模型 | 无需维护 |

#### 2. 代码变更

##### 2.2 新增 DeepSeekModelClient 类

**新增代码** (两个版本都添加):
```python
class DeepSeekModelClient:
    def __init__(self, model="deepseek-chat", api_key=None, temperature=0.2, top_p=0.9, timeout=300):
        self.model = model
        self.temperature = temperature
        self.top_p = top_p
        self.timeout = timeout
        self.api_key = api_key or os.environ.get('DEEPSEEK_API_KEY')
        if not self.api_key:
            raise RuntimeError("DEEPSEEK_API_KEY not found in environment variables")
        
        from openai import OpenAI
        self.client = OpenAI(
            api_key=self.api_key,
            base_url="https://api.deepseek.com"
        )

    def complete(self, prompt, max_new_tokens):
        try:
            response = self.client.chat.completions.create(
                model=self.model,
                messages=[
                    {"role": "user", "content": prompt},
                ],
                temperature=self.temperature,
                top_p=self.top_p,
                max_tokens=max_new_tokens,
                stream=False
            )
            return response.choices[0].message.content
        except Exception as exc:
            raise RuntimeError(f"DeepSeek API request failed: {exc}")
```
