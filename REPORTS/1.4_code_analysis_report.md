```python
def note_tool(self, name, args, result):
    memory = self.session["memory"]
    path = args.get("path")
    # 记录文件操作
    if name in {"read_file", "write_file", "patch_file"} and path:
        self.remember(memory["files"], str(path), 8)
    # 记录工具调用摘要
    note = f"{name}: {clip(str(result).replace('\n', ' '), 220)}"
    self.remember(memory["notes"], note, 5)
```

**remember() 方法** (Line 271-277):
```python
@staticmethod
def remember(bucket, item, limit):
    if not item:
        return
    if item in bucket:
        bucket.remove(item)
    bucket.append(item)
    del bucket[:-limit]
```

### 2.3 Memory 读取机制

**memory_text() 方法** (Line 376-385):
```python
def memory_text(self):
    memory = self.session["memory"]
    notes = "\n".join(f"- {note}" for note in memory["notes"]) or "- none"
    return "\n".join([
        "Memory:",
        f"- task: {memory['task'] or '-'}",
        f"- files: {', '.join(memory['files']) or '-'}",
        "- notes:",
        notes,
    ])
```

### 2.4 现有 Memory 机制的局限性

| 限制点 | 描述 | 影响 |
|--------|------|------|
| **容量固定** | files 最多8个，notes 最多5个 | 长对话场景下关键信息可能丢失 |
| **无压缩** | notes 直接存储原始摘要 | token 消耗大 |
| **无约束保留** | 没有专门机制保留用户约束 | 约束容易被遗忘 |
| **无阶段感知** | 所有历史同等对待 | 无法识别工作阶段 |
| **无智能筛选** | 所有工具调用同等记录 | 噪音数据过多 |

### 3.2 历史文本生成与裁剪

**history_text() 方法** (Line 390-417):
```python
def history_text(self):
    history = self.session["history"]
    if not history:
        return "- empty"

    lines = []
    seen_reads = set()
    recent_start = max(0, len(history) - 6)

    for index, item in enumerate(history):
        recent = index >= recent_start

        # 工具调用去重: 合并同一文件的重复读取
        if item["role"] == "tool" and item["name"] in ("write_file", "patch_file"):
            path = str(item["args"].get("path", ""))
            seen_reads.discard(path)
        if item["role"] == "tool" and item["name"] == "read_file" and not recent:
            path = str(item["args"].get("path", ""))
            if path in seen_reads:
                continue
            seen_reads.add(path)

        # 长度裁剪策略
        if item["role"] == "tool":
            limit = 900 if recent else 180  # 近期900字符，历史180字符
            lines.append(f"[tool:{item['name']}] {json.dumps(item['args'], sort_keys=True)}")
            lines.append(clip(item["content"], limit))
        else:
            limit = 900 if recent else 220  # 近期900字符，历史220字符
            lines.append(f"[{item['role']}] {clip(item['content'], limit)}")

    return clip("\n".join(lines), MAX_HISTORY)  # MAX_HISTORY = 12000
```

### 3.3 Transcript 裁剪策略总结

| 策略 | 近期记录 (最后6条) | 历史记录 |
|------|-------------------|----------|
| 工具输出 | 900 字符 | 180 字符 |
| 对话内容 | 900 字符 | 220 字符 |
| 读取去重 | 无 | 同一文件只保留最后一次读取 |
| 全局截断 | 12000 字符上限 | 12000 字符上限 |

### 3.4 现有 Transcript 处理的问题

1. **暴力截断**: 使用 `clip()` 简单截断，可能切断语义单元
2. **固定窗口**: 只保留最后6条为"近期"，其余都视为历史
3. **无语义理解**: 无法识别关键决策、约束条件
4. **工具输出噪音**: 测试输出、日志等大量无用信息未被过滤

## 6. Memory Manager 设计建议

### 6.1 推荐的 3 个 Feature 实现

基于代码分析，推荐实现以下三个 feature：

#### Feature 1: Rolling Summary (滚动摘要)
- **目标**: 在长对话中自动生成摘要，防止历史记录过长
- **实现位置**: `ask()` 方法中的 `ask()` 主循环
- **触发条件**: history 长度超过阈值（如 50 条记录）
- **摘要内容**: 任务进展、已完成步骤、当前问题

#### Feature 2: Pinned Facts / Constraints (关键约束保留)
- **目标**: 确保用户约束（如"不要修改某文件"）在整个对话中被记住
- **实现位置**: `note_tool()` 方法
- **识别方式**: 关键词匹配（"不能"、"不要"、"必须"、"保持"等）
- **存储位置**: 独立的 pinned_constraints 列表

#### Feature 3: Tool Output Compression (工具输出压缩)
- **目标**: 智能压缩大量测试输出、日志等噪音数据
- **实现位置**: `history_text()` 方法
- **压缩策略**:
  - 识别测试输出模式（PASSED/FAILED 统计）
  - 识别错误日志
  - 对重复内容进行折叠

### 6.2 不推荐的 Feature

**Task-State Segmentation (任务状态分段)** - 不推荐原因：
- 当前架构缺乏明确的阶段边界定义
- 实现复杂度高，需要重新设计 history 结构
- 收益相对较低，其他三个 feature 可以解决大部分问题

### 6.3 建议的数据结构

```python
self.session["memory"] = {
    "task": "",                          # 原始任务描述
    "summary": "",                        # 滚动摘要 (新增)
    "pinned_constraints": [],             # 固定约束列表 (新增)
    "files": [],                          # 最近文件
    "notes": [],                          # 常规笔记
    "compression_stats": {                # 压缩统计 (新增)
        "total_tools": 0,
        "compressed_tools": 0,
        "compression_ratio": 0.0
    }
}
```

## 8. 风险评估

### 8.1 技术风险

| 风险 | 等级 | 缓解策略 |
|------|------|---------|
| LLM 摘要质量不稳定 | 中 | 提供示例模板，引导 LLM 生成结构化摘要 |
| 约束误识别 | 中 | 使用保守的关键词列表，允许用户手动标记 |
| 压缩丢失关键信息 | 高 | 保留所有错误信息，仅压缩重复/噪音 |

### 8.2 兼容性风险

- **会话恢复**: 新版本 memory 结构需要向后兼容旧版本
- **跨版本迁移**: 提供数据迁移脚本
